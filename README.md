Internal Linux Security Assessment

This project simulates an internal Linux security assessment and demonstrates my understanding of real‑world penetration‑testing methodology.
It highlights my ability to organize findings, document each phase clearly, and follow a professional security‑assessment workflow.

📌 Project Overview
This repository walks through the major phases of a Linux security assessment:

Reconnaissance

Enumeration

Exploitation

Privilege Escalation

Reporting

Each folder contains notes, commands, and outputs collected during the assessment process.

Folder Structure
1. Reconnaissance
Initial scanning and service discovery.
Includes Nmap results and early observations.

2. Enumeration
Detailed inspection of identified services, users, directories, and potential attack surfaces.

3. Exploitation
  Notes and steps taken to exploit discovered vulnerabilities or misconfigurations.

4. Privilege Escalation
Techniques used to elevate privileges after gaining initial access.

5. Report
 A summary of findings, risks, and recommendations.

🛠 Tools Used

Nmap

Linux command‑line utilities

Manual enumeration techniques

Basic exploitation workflow
